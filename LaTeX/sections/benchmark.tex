\section{Simulator Benchmarking}

During all the test development and demonstration, I have been using a laptop with the following specifications:

\begin{itemize}
    \item CPU: AMD Ryzen™ 5 7530U
    \item RAM: 24 GB DDR4
    \item GPU: Integrated AMD Radeon™ Graphics
    \item OS: Ubuntu 24.04.4 LTS
\end{itemize}


Simulator was always running in Unity Editor, which is not the most efficient way to run it, but it was the most convenient for development and testing. Nevertheless, Unity app can be built and run as a standalone executable, which should provide better performance. This I will test in the futurer sections, but for now I will just provide some benchmark results from running the simulator in the editor.

\subsection{n-UAV's Performance Test}
In this test I wanted to check how many drones can be simulated in real time in the Unity Editor. I have started with no drones and then I have been increasing the number of drones until I reached the limit of real time simulation (at least 30 FPS). The results are summarized in the figure below: \ref{fig:performance_test}.

In the begining system was limited to 90 FPS, witch is refresh rate of laptop screen. After around 40 drones simulated at the same time, the performance started to drop. This is a most likely due to Unity Engine CPU scheduler, which is not designed to handle such a high number of objects at the same time. This could be solved by making code less universal and more optimized for specific use case, but I have decided to make it modular and easy to understand. This lets me or other users to easily modify and extend the simulator. With the current implementation, simulator can run in real time with up to 100 drones, witch is more than enough for most of the use cases. For example, if we want to test a control algorithm based on Reinforcement Learning, we can run multiple instances of the simulator in parallel, each with a different number of drones, and then we can use the data from all the instances to train our algorithm.
